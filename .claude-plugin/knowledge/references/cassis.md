# Cassis — SaaS minimal com cards interativos

**URL:** https://cassis.framer.website/
**Vibe:** clean, modular, SaaS premium. Hero curto, features como cards "vivos" (calendário, gráficos, drop-zone), depoimentos com aspas oversized.

## Padrões úteis

1. **Hero curto com pill de update no topo.** Acima do headline, um pill clicável tipo "New Spring Update" linka pra changelog. Cria movimento sem competir com o CTA.
2. **Features como mini-screenshots vivos.** Em vez de ícone + texto, cada feature é um card com UI real do produto: calendário com eventos, gráfico de performance, drop-zone de upload. Cria desejo pelo "como é por dentro".
3. **Bento grid de 3-4 cards de tamanhos diferentes.** Um card grande (hero feature) + 2-3 menores. Quebra a monotonia da grid uniforme.
4. **Headline oversized colado nas palavras.** "Wehelppeopleturnordinarycontent…" sem espaços — tipografia como arte. (No Umbler DS, replicar com `text-display` + tracking apertado, não literalmente colando.)
5. **"3 steps to launch" com UI mockada em cada step.** Step 1: drop zone. Step 2: tweaking goals. Step 3: results dashboard. Cada passo *mostra* o produto, não só conta.
6. **Testimonials com aspas H2-sized.** A frase do cliente é maior que o nome — inverte a hierarquia padrão. A frase vende, não o autor.

## Estrutura das seções (ordem)

1. Hero (pill de update → headline → subtext curto → video CTA)
2. Logo cloud "Trusted by"
3. Features (4 cards bento, UI viva)
4. Quote oversized do CEO/founder
5. Feature deep-dive (3 blocos com screenshot grande + texto curto ao lado)
6. Key capabilities (grid 2x2 de 4 mini-features)
7. Get Started (3 steps com mockup por step)
8. Testimonials (5 cards com aspas H2)
9. FAQ (accordion)
10. CTA final

## Copy moves notáveis

- **Headlines descritivos, não slogans.** "Built for clarity, built for speed" — descreve a sensação, não promete benefício mágico.
- **Eyebrows funcionais.** "Features", "Performance Insights", "Smart Organization" — direto, sem floreio.
- **Subtexts em uma linha.** "Easy-to-read metrics that help you understand results." — verbo + objeto + benefício.

## O que evitar copiar

- Tipografia sem espaços nas headlines (Umbler usa display com kerning normal)
- Imagens stock de pessoas sorrindo em escritório (Umbler tem brand própria)
