# Fusion AI — Produto AI/Agente com narrativa de capacidade

**URL:** https://fusionai.framer.website/
**Vibe:** SaaS de AI agents/workflow automation. Hero promete superpoder ("Supercharge"), features mostram integrações reais (logos), prova social numérica ("150,000+ users").

## Padrões úteis

1. **Headline com verbo "Supercharge"/"Power up".** Promete amplificação, não substituição. "Supercharge your AI workflows" — você já tem o workflow, a gente potencializa.
2. **Prova social numérica grande no hero.** "150,000+ users worldwide" — número como pilar do hero, não buried no rodapé.
3. **Integrations row como prova visual.** Slack, HubSpot, Zendesk, etc. como logos — mostra ecosystem. Substitui ou complementa o logo cloud genérico.
4. **Features em 4 grupos com nomenclatura forte.** "Seamless Integrations", "Conversational Actions", "Visual Workflow Designer", "Multi-Channel Automation". Cada nome promete uma capacidade clara.
5. **Deep-dive de 3 features depois.** Após o "quick view" de 4, abre 3 features em detalhes longos com screenshot. Camada de discovery: scan rápido + leitura aprofundada.
6. **Onboarding "select → connect → automate".** Verbo no imperativo, 3 passos só, cada um é uma ação clara do usuário.
7. **FAQ como objeções específicas.** "Is it secure?", "What support is available?" — endereça medos concretos, não dúvidas genéricas.

## Copy moves notáveis

- **Verbos de amplificação.** "Supercharge", "Power up", "Unlock". O usuário não é trocado, é amplificado.
- **Trigger-Action-Result.** "Connect applications, establish triggers, and let AI manage tasks automatically without coding expertise." — fórmula de 3 partes que define o produto em uma frase.
- **"Without [pain]" closer.** "without extensive development time", "without coding expertise". Promessa de remover atrito.

## Quando usar este padrão

Produtos de **automação, AI, integração**: o usuário tem capacidade técnica e quer escalar. O Umbler Talk se encaixa parcialmente — o agente AI "amplifica" o atendimento humano em vez de substituir.
