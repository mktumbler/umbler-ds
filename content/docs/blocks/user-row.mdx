## Clicável — Modal de seleção

Quando a linha inteira é a ação (abrir conversa, selecionar contato), use `asChild` envolvendo a linha com `<a>`, `<button>` ou `<Link>`. O bloco adota o tag do filho, herdando estilo + hover + focus-visible. A acessibilidade vem de graça (role correto, navegação por teclado, ring no foco).

```tsx
<UserRow asChild name="Jana" avatar={{ name: 'Jana Silva' }} subtitle="…">
  <Link href="/contacts/jana" />
</UserRow>
```

## Componentes usados

- [`Avatar`](/docs/components/avatar) — props passadas direto via `avatar={...}`
- [`Checkbox`](/docs/components/checkbox) — quando `selectable`
- [`Tag`](/docs/components/tag) — para as etiquetas (Proposta, Sales, etc.). É o pill arredondado padrão do Umbler Talk. Não confunda com `Badge shape="tag"`, que é o estilo retangular/uppercase usado em changelog/roadmap.

## Boas práticas

- **Avatar sempre que possível.** Em listas de pessoas, o reconhecimento visual depende dele. Sem avatar a linha vira "lista de strings".
- **Não combine `selectable` com `asChild` na mesma linha.** Se a linha é clicável e selecionável simultaneamente, fica ambíguo o que o clique faz. Escolha um modelo.
- **Tags são contexto, não decoração.** Use 1–3 tags por linha; mais que isso vira ruído. Se precisa exibir muita info, é card, não row.
- **`actions` reserva à direita.** Não amontoe vários ícones — se precisa de 2+ ações, use Dropdown (kebab menu).
