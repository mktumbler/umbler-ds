## Quando usar

- **Lista/tabela vazia** — "Nenhum contato aqui ainda"
- **Estado não conectado** — "Seus emails não estão conectados à Umbler"
- **Onboarding bloqueante** — "Para usar sites você vai precisar de um domínio"
- **Resultado de busca vazio** — "Nada encontrado"

Não use pra **promoção de feature** (compra de plano, ative algo pago). Pra isso é o `<MarketingBanner>` (split com imagem + benefícios).

## Onde aparece no produto

| Tela | Variante |
|---|---|
| Console → DNS | Email não conectado — ilustração + título + 2 CTAs |
| Modal Subdomínio | Onboarding bloqueante — ícone + título + input + CTA |
| Talk → listagens vazias | Sem conversas / sem contatos — info-only |
| Talk → busca | Resultado vazio — pequeno, info-only |

## Boas práticas

- **Toda EmptyState diz "o que" e "o que fazer".** Título nomeia o estado, descrição explica o porquê, ação resolve. Sem ação = sem urgência (ex: lista naturalmente vazia que vai se preencher).
- **CTA primária resolve o estado, secundária oferece alternativa.** "Conectar DNS Umbler" (primária) + "Usar DNS de terceiros" (secundária). Nunca 2 primárias.
- **Não use verde como CTA.** Verde é estado. Veja [colors.mdx](/docs/foundations/colors).
- **Tom evita culpar o usuário.** "Nenhuma conversa por aqui" é melhor que "Você não tem conversas". Vazio é estado do sistema, não falha da pessoa.
- **Tamanho responde ao espaço, não à importância.** EmptyState de tabela é `sm`; de página inteira é `lg`. Não é "este é mais importante então grande".
