**Blocks** são organismos: pedaços de UI grandes o suficiente pra ter identidade própria (uma seção de hero, uma tabela de preços, uma linha de contato), montados a partir dos [Components](/docs/components/button). São o atalho para montar uma página rápido sem reinventar.

Todos são **instaláveis avulso** — caem em `components/blocks/` e puxam os componentes que usam:

```bash
npx shadcn@latest add https://umbler-ds.vercel.app/r/hero-block
```

## Landing / Marketing

A maior parte de uma LP Umbler sai daqui. Combine na ordem da [sequência de conversão](/docs/marketing/conversion).

| Bloco | Para que serve | Slug |
|---|---|---|
| [HeroBlock](/docs/blocks/hero-block) | Abertura da página: eyebrow, headline, subcopy, CTAs e mídia | `hero-block` |
| [PricingTable](/docs/blocks/pricing-table) | Planos lado-a-lado com destaque de plano popular | `pricing-table` |
| [CTABanner](/docs/blocks/cta-banner) | Faixa de conversão (premium/glow/simple) pra fim de seção | `cta-banner` |
| [LogoCloud](/docs/blocks/logo-cloud) | Logos de clientes (social proof) em grade ou marquee infinito | `logo-cloud` |
| [Section Blocks](/docs/blocks/section-blocks) | FAQ (reduz objeção), StatGrid (prova com números), Testimonial (voz do cliente) | `faq-section` · `stat-grid` · `testimonial-block` |
| [FeatureCardGrid](/docs/blocks/feature-card-grid) | Grade de features com ícone + título + descrição | `feature-card-grid` |

## Produto

Composições recorrentes nas telas de produto (Talk, Hosting, Console).

| Bloco | Para que serve | Slug |
|---|---|---|
| [DataListPage](/docs/blocks/data-list-page) | Esqueleto de tela de listagem: header + toolbar + tabela | `data-list-page` |
| [UserRow](/docs/blocks/user-row) | Linha de contato/usuário: avatar + texto + ações | `user-row` |
| [FormPanel](/docs/blocks/form-panel) | Painel de formulário em Card (header + campos + ações) | `form-panel` |
| [EmptyState](/docs/blocks/empty-state) | Estado vazio: ícone + título + descrição + ação | `empty-state` |

## Princípios

- **Bloco não reinventa componente nem cria token.** Compõe `<Button>`, `<Card>`, `<Avatar>` e tokens existentes. Guardado por `scripts/audit-antipatterns.mjs`.
- **Bloco de produto segue a regra do 3** (Polaris/Primer): só vira bloco o padrão que aparece em 3+ telas diferentes. Blocos de landing seguem a taxonomia padrão de seções de LP.
- **Dados via props** — nenhum texto fica hardcoded; o bloco recebe conteúdo de fora.
