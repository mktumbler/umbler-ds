## Quando usar

- **Criar entidade** — novo grupo, novo filtro, novo contato, nova campanha
- **Editar entidade** — atualizar configurações de algo já existente
- **Modais bloqueantes com 1 ação principal** — "informe um domínio para continuar"

Não use pra **filtros de listagem inline** (esse é o `<DataToolbar>`). Não use pra **formulários longos com seções** (esse é um pattern de página, não bloco).

## Onde aparece no produto

| Tela | Variante |
|---|---|
| Console → Novo filtro de email | Header card + form com selects/inputs + CTA verde full-width |
| Console → Novo grupo de email | Header card + Nome/Email/Membros + CTA full-width |
| Modal Subdomínio | Form enxuto com 1 input + CTA full-width |

Quando o submit dividir espaço com uma ação secundária (cancelar, voltar, deletar), use `footerFullWidth={false}`. As ações alinham à direita.

## Boas práticas

- **Submit full-width = 1 ação principal.** Se tem cancelar/voltar/deletar, use `footerFullWidth={false}` pra colocar as 2 ações lado-a-lado à direita.
- **Não verde como CTA.** Verde é estado. O CTA do form é `<Button>` (azul brand).
- **Largura máxima.** O FormPanel já vem com `max-w-2xl mx-auto`. Pra formulários mais largos (ex: 2 colunas reais), passe `className="max-w-4xl"`.
- **InputLabel sempre acima do campo.** É um par tipográfico, não label flutuante — colocar inline confunde acessibilidade.
- **Não aninhe FormPanel dentro de FormPanel.** Se você precisa de "form dentro de form", o pattern é seções de página, não cards aninhados.
