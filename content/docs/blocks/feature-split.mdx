A maior parte das LPs da Umbler é uma sequência de seções problema/solução. O `FeatureSplit` dá o esqueleto: à esquerda, o argumento (eyebrow, título, parágrafo, bullets, CTA); à direita, o visual (mock, imagem, ilustração). O visual é um slot — você passa qualquer `ReactNode`.

## API

```tsx
import { FeatureSplit } from '@/components/blocks/feature-split';

<FeatureSplit
  eyebrow="Times"
  title="Quando um funcionário tira férias…"
  body="O Umbler Talk distribui as conversas entre o time."
  bullets={['Filas por equipe', 'Round-robin automático']}
  cta={{ label: 'Experimente grátis', href: '/signup' }}
  visual={<img src="/mock-teams.png" alt="Distribuição entre times" />}
  reverse
/>
```

## Boas práticas

- **Alterne `reverse`** entre seções consecutivas — sequência sem alternância vira coluna monótona
- **Visual conta uma história** — mock do produto, screenshot ou ilustração concreta. Evite imagem-stock genérica
- **Bullets curtos** — 3-5 itens, cada um cabe em uma linha de leitura
- Para muitas seções na mesma página (6+ no Umbler Talk), considere agrupá-las sob um título de "categoria" para o leitor não se perder
