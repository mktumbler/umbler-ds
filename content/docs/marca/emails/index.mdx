O DS inclui templates de email desenvolvidos com [react-email](https://react.email/),
que compilam para HTML email-safe com CSS inline — compatíveis com Gmail, Outlook,
Apple Mail e os principais clientes.

## Por que react-email?

Emails têm constraints diferentes do web: **sem CSS variables, sem flexbox confiável,
sem Tailwind** — o output precisa ser HTML de tabelas com estilos inline. O react-email
resolve isso mantendo o paradigma de componentes React que o DS já usa.

Os **tokens de cor** são os mesmos do DS (brand-500 = `#1a5cff`, neutral-950 = `#030614`, etc.),
apenas aplicados como valores literais no CSS inline.

valores prontos pra CSS inline em qualquer ESP.

## Download e uso

Cada template tem um botão **⬇ Download HTML** que baixa o HTML final pronto para
importar em Mailchimp, HubSpot, SendGrid, Amazon SES ou qualquer ESP.

Para uso programático com a API do react-email:

```ts
import { render } from '@react-email/render';
import WelcomeEmail from '@/emails/templates/welcome';

const html = await render(<WelcomeEmail firstName="Ana" />);
// html = string com o HTML completo, CSS inline, pronto para envio
```

## Convenções

- **Fundo escuro** — todos os templates usam o fundo `#030614` (neutral-950) da marca
- **System font stack** — `-apple-system, "Segoe UI", Roboto, ...`. Renderiza SF Pro no macOS/iOS, Segoe UI no Windows, Roboto no Android. Inter foi descartada porque clientes de email (Outlook desktop, Gmail mobile, Yahoo) ignoram `@font-face` e caem em Times New Roman
- **Cores hardcoded** — necessário; clientes de email ignoram CSS custom properties
- **Tokens espelhados** em `emails/components/layout.tsx` como objeto `t` — altere lá e todos os templates herdam (inclusive `t.fontStack`)
- **Logo via SVG** — funciona em Apple Mail, Gmail e clients modernos. Para garantir Outlook desktop, considere servir um PNG `@2x` substituindo a URL do `<Img>` no `EmailLayout`
