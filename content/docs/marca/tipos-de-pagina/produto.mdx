## Objetivo

Visitante sai com:
- Entendimento claro do **que o produto faz** e como faz
- Confiança de que **funciona pra ele** (caso, persona, segmento)
- Próxima ação óbvia: **iniciar trial** ou **agendar demo**

## Seções opcionais

- **Comparison Table** — Umbler vs. alternativa
- **Testimonial Carousel** — depoimentos com especificidade

- **Problema → Solução** (pares) — para LPs onde a dor não está óbvia

## Padrões narrativos por produto

### Agente IA (Solution Aware — comparando IAs)

```
Hero → Logo Cloud → Feature Cards (3-col) → Feature Showcase →
Use Cases → Customização (Steps 01/02/03) → Multi-feature list →
Comparação (IA vs Chatbot) → Integrações → Pricing → FAQ
```

### Chatbot (Problem Aware — sabe que precisa, descobrindo)

```
Hero → Screenshot → Feature Cards → Indústrias/Segmentos →
Integrações → Pricing → FAQ
```

### CRM WhatsApp (Unaware do produto, aware da dor)

```
Hero → Logo Cloud → Problema/Solução (pares) → Feature Grid (6) →
Pricing → FAQ
```

<Metodologia title="Por que cada produto tem sequência própria">

Os 3 produtos Umbler têm sequências diferentes baseadas na maturidade da persona:

- **Agente IA:** a persona é Solution Aware (comparando IAs). Diferenciação aparece cedo (Feature Cards 3-col), comparação aparece tarde (já comprou a ideia).
- **Chatbot:** persona Problem Aware. LP mais curta; Indústrias/Segmentos no meio porque o leitor quer ver "isso serve pra mim?".
- **CRM WhatsApp:** persona Unaware do produto, aware da dor. Pares de Problema/Solução logo cedo — agitate problem antes de qualquer feature, porque a persona ainda está reconhecendo a dor.

</Metodologia>

## Tom

Direto, com números, foco em **outcomes operacionais**. Verbos concretos no Feature Showcase: responde, qualifica, organiza, distribui, agenda. Sem jargão técnico (a persona é gestor, não dev).

## Anti-patterns

- Feature Showcase com nomes de módulos ("Módulo de Qualificação") em vez de benefícios ("Leads qualificados de verdade")
- FAQ genérico ("Quanto custa?") — perguntas devem espelhar objeções reais ("E se meu time não usar?")
- Sem Social Proof antes do 3º scroll
- CTA "Saiba mais" em qualquer lugar
- Falar do "como funciona internamente" antes do "o que muda pra você"
