A paleta Umbler é ancorada no **azul profundo** e em uma escala neutra densa, pensada para experiências digitais premium em dark mode (modo padrão).

Todos os tokens são exportados como CSS custom properties (`--color-*`) e ficam disponíveis como classes utilitárias do Tailwind (`bg-brand-500`, `text-neutral-100`, etc.).

## Primary — Umbler Blue

O azul é a cor de marca. Use **500** como cor principal (botões primários, links, focus rings) e **400** para hover states.

## Cor para ação vs. estado

> **Verde é estado, não ação.** Use `success-*` em badges, toasts, status pills e ícones de feedback positivo. Para chamar ação (salvar, confirmar, enviar, publicar), use sempre `Button variant="primary"`.

Vale pra paridade entre produtos e pra acessibilidade — daltônicos e leitores de tela leem a **forma + texto** do CTA, não a cor.

### Intenção → componente

| Intenção | Componente |
|---|---|
| Ação primária (qualquer verbo: salvar, criar, confirmar, enviar, publicar) | `Button variant="primary"` |
| Ação secundária (cancelar, voltar) | `Button variant="secondary"` ou `ghost` |
| Ação destrutiva (excluir, remover) | `Button variant="danger"` |
| Estado "deu certo" (pago, ativo, conectado, concluído) | `Badge variant="success"` / toast `success` |
| Estado de atenção (pendente, expirando) | `Badge variant="warning"` / toast `warning` |
| Estado de erro (falha, recusado) | `Badge variant="error"` / toast `error` |

### Faça / Não faça

<VariantGrid columns={2}>
  <VariantCard label="✓ Faça" hint="Ação primária em azul brand">
    <Button>Confirmar pagamento</Button>
  </VariantCard>
  <VariantCard label="✗ Não faça" hint="Verde como CTA dilui o sinal de ação">
    {/* audit-ignore: button-hand-roll — ilustração de misuse para a regra "verde nunca é CTA" */}
    <Button className="!bg-success-500 hover:!bg-success-600 opacity-60 line-through decoration-2">
      Confirmar pagamento
    </Button>
  </VariantCard>
</VariantGrid>

O verde fica reservado pra _comunicar_ sucesso depois — `Badge variant="success"`, toast de sucesso, check-icon. Nunca no botão que dispara a ação.

## Brand Gradient

O gradient de marca é usado em hero sections, destaques e elementos decorativos pontuais. Em **dark mode**, ele é o linear azul→salmão (`#8BBCF8 → #F8C4B8`). Em **light mode**, o eyebrow cai pra azul sólido `#1A5CFF` (mantém contraste em superfícies claras).

<ComponentPreview title="Brand gradient (dark)" background="dark">

    style={{

      maxWidth: '480px',
      height: '120px',
      borderRadius: '0.625rem',
      background: 'linear-gradient(90deg, #8BBCF8 0%, #F8C4B8 100%)',
    }}
  />
</ComponentPreview>

```css
--gradient-brand: linear-gradient(90deg, #8BBCF8 0%, #F8C4B8 100%);
```

```tsx
<div className="bg-[image:var(--gradient-hero)]">...</div>
```

## Shadow CTA

Sombra colorida no azul da marca, pensada pra **CTAs primários de landing** (CTAs de produto continuam com `shadow-sm`/`shadow-md`). Em dark a sombra é mais intensa para compensar a luminosidade do fundo.

```tsx
<Button className="shadow-cta">Experimente por 7 dias</Button>
```

## Usando no código

```tsx
// Como classe utilitária Tailwind
<div className="bg-brand-500 text-neutral-50">Hello Umbler</div>

// Como CSS variable
<div style={{ background: 'var(--color-brand-500)' }}>Hello Umbler</div>
```
