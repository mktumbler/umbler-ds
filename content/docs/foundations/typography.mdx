## Famílias

| Família | Uso | Token |
|---|---|---|
| P22 Mackinac | Títulos (Display, H1–H3) | `--font-heading` |
| Inter | UI, body, labels | `--font-sans` |
| JetBrains Mono | Código, terminais, dados estruturados | `--font-mono` |

## Tokens

Cada step da escala tem um token semântico Tailwind v4:

```css
/* ── Display — impacto e marketing ── */
text-display-2xl  /* 128px  lh 0.95  ls -0.40rem */
text-display-xl   /*  96px  lh 1.00  ls -0.20rem */
text-display-lg   /*  72px  lh 1.05  ls -0.15rem */
text-display      /*  56px  lh 1.05  ls -0.10rem */

/* ── Headings — conteúdo estruturado ── */
text-h1           /*  48px  lh 1.10  ls -0.10rem */
text-h2           /*  36px  lh 1.20  ls -0.05rem */
text-h3           /*  28px  lh 1.25 */
text-h4           /*  22px  lh 1.35 */

/* ── Body ── */
text-body-lg      /*  18px  lh 1.60 */
text-body         /*  16px  lh 1.60 */
text-body-sm      /*  14px  lh 1.50 */
text-caption      /*  12px  lh 1.40 */
```

## Elemento vs. classe — regra de ouro

**O elemento `<h2>` sempre recebe `text-display` (56 px).** Sem exceções.

A classe `text-h2` (36 px) existe para elementos que *visualmente lembram* um heading mas não são `<h2>` — stats em cards, títulos de painel de app, labels compostos. Nunca coloque `text-h2` num elemento `<h2>`.

| Elemento / contexto | Token correto |
|---|---|
| `<h2>` (qualquer contexto) | `text-display` (56 px) |
| `<h1>` hero principal | `text-display-lg` (72 px) |
| `<h1>` splash | `text-display-xl` / `text-display-2xl` |
| Stat, label de card, painel de app | `text-h2` (36 px) num `<p>` ou `<span>` |

O componente `SectionHeader` aplica `font-heading text-display text-foreground text-balance` automaticamente — prefira-o a escrever o `<h2>` à mão.
