Compound semântico (`<Table>`, `<TableHeader>`, `<TableBody>`, `<TableRow>`, `<TableHead>`, `<TableCell>`, `<TableCaption>`, `<TableFooter>`) sobre elementos nativos do HTML. Para listas com filtros, ordenação e seleção, embrulhe com TanStack Table — os primitivos daqui cuidam só do visual e da acessibilidade base.

## API — Table

| Prop          | Tipo      | Default | Descrição                                        |
| ------------- | --------- | ------- | ------------------------------------------------ |
| `zebra`       | `boolean` | `false` | Alterna o fundo das linhas pares.                |
| `dense`       | `boolean` | `false` | Padding compacto.                                |
| `bordered`    | `boolean` | `false` | Bordas internas verticais entre colunas.         |
| `interactive` | `boolean` | `false` | Hover destaca a linha; cursor pointer.           |

As variantes são propagadas via Context — define no `<Table>` e os filhos consomem automaticamente.

## Estado selecionado

Use `data-state="selected"` no `<TableRow>` para marcar linhas com checkbox/seleção:

```tsx
<TableRow data-state={isSelected ? 'selected' : undefined}>
  ...
</TableRow>
```

## Acessibilidade

- Elementos nativos: `<table>`, `<thead>`, `<tbody>`, `<th>`, `<td>` — leitores anunciam estrutura corretamente.
- `<TableCaption>` vira `<caption>` semântico (resumo da tabela).
- Em listas clicáveis com `interactive`, prefira embrulhar o conteúdo do `<TableCell>` em `<a>` ou `<button>` real.

## Uso

```tsx
import {
  Table, TableHeader, TableBody, TableRow, TableHead, TableCell,
} from '@/components/ui/table';

<Table zebra>
  <TableHeader>
    <TableRow>
      <TableHead>Nome</TableHead>
      <TableHead>Plano</TableHead>
    </TableRow>
  </TableHeader>
  <TableBody>
    {data.map((row) => (
      <TableRow key={row.id}>
        <TableCell>{row.nome}</TableCell>
        <TableCell>{row.plano}</TableCell>
      </TableRow>
    ))}
  </TableBody>
</Table>
```
