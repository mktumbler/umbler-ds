Use `Tooltip` para descrever ações abreviadas (ícones, botões sem rótulo) ou explicar um termo. Mantenha o texto curto — para conteúdo rico ou clicável, prefira `Popover`.

Envolva sua árvore (ou o trecho que usa tooltips) em `TooltipProvider` uma única vez.

## Conteúdo rico

Apesar do `Tooltip` ser pensado para texto curto, ele aceita qualquer ReactNode quando necessário.

## Acessibilidade

- `TooltipTrigger` deve envolver um elemento focusável (botão, link). Use `asChild` para herdar o elemento filho ao invés de inserir um wrapper.
- O tooltip abre no `hover` e no `focus` do trigger, e fecha no `blur` ou `Escape`.
- Para um ícone-only, lembre de fornecer `aria-label` no botão — o texto do tooltip não é exposto como nome acessível.

## Uso

```tsx
import {
  Tooltip,
  TooltipContent,
  TooltipProvider,
  TooltipTrigger,
} from '@/components/ui/tooltip';

<TooltipProvider delayDuration={150}>
  <Tooltip>
    <TooltipTrigger asChild>
      <Button variant="ghost" aria-label="Editar">
        <PencilSimple size={16} />
      </Button>
    </TooltipTrigger>
    <TooltipContent side="top">Editar item</TooltipContent>
  </Tooltip>
</TooltipProvider>
```
