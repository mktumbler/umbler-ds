A `Tag` é como um Badge, mas inclui um botão de remoção. Use em campos de multi-seleção, filtros ativos e listas de palavras-chave editáveis pelo usuário.

## Campo de tags

Para um campo de formulário que aceita múltiplas tags, use o componente dedicado [**TagInput**](/docs/components/tag-input) — dimensionado igual ao `Input`, com estados de validação.

## API

| Prop        | Tipo                                                           | Default     | Descrição                                        |
| ----------- | -------------------------------------------------------------- | ----------- | ------------------------------------------------ |
| `variant`   | `'brand' \| 'success' \| 'warning' \| 'error' \| 'neutral'` | `'neutral'` | Cor semântica da tag                             |
| `size`      | `'sm' \| 'md'`                                                | `'md'`      | Tamanho: sm = 11px, md = 12px                   |
| `onRemove`  | `() => void`                                                   | —           | Callback ao clicar no ×. Se ausente, sem botão  |
| `disabled`  | `boolean`                                                      | `false`     | Desabilita o botão de remoção                   |
| `className` | `string`                                                       | —           | Classes extras                                   |

## Uso

```tsx
import { Tag } from '@/components/ui/tag';

// Tag simples (sem remoção)
<Tag variant="brand">Next.js</Tag>

// Tag removível
<Tag variant="brand" onRemove={() => removeTag('Next.js')}>
  Next.js
</Tag>

// Desabilitada
<Tag variant="neutral" onRemove={() => {}} disabled>
  Locked
</Tag>
```
