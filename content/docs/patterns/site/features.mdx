Seções de features comunicam o valor do produto através de estrutura visual clara. Três padrões cobrem a maioria dos contextos: **grid** para listar muitas funcionalidades, **split** para destacar uma com profundidade, e **bento** para criar hierarquia visual com impacto.

## Grid

Ideal para apresentar 4–6 funcionalidades em pé de igualdade. O grid com divisor visual (`gap-px` + fundo no container) cria separação sem linhas duras.

<FeatureGridSection />

## Split

Um feature em destaque com texto detalhado à esquerda e visual representativo à direita. Use quando um benefício merece mais contexto — CRM, IA, automação.

<FeatureSplitSection />

## Bento

Grid assimétrico que cria hierarquia visual: o card maior ancora a narrativa, os menores complementam. Funciona melhor sobre fundo escuro para destacar o efeito aurora.
