LP de Use Case mergulha em **UM resultado concreto**. Não vende o produto inteiro — vende uma aplicação específica que resolve uma dor pontual. É o "como Umbler faz X" detalhado.

## Objetivo

Visitante sai com:
- Entendimento **funcional** do como o caso funciona (passos, screenshots, exemplos)
- **Convicção** de que o resultado prometido é alcançável
- Próximo passo claro: **trial** ou **demo dedicada**

## Seções opcionais

- **Stats Counter** — métricas do antes/depois ("70% dos leads respondidos em menos de 3s")
- **FAQ específico** do use case
- **Testimonial Carousel** — cliente que aplicou o caso
- **Steps Section** — 3 passos pra ativar o caso

## SEO

- **Meta title:** `[Use Case] — Como [resultado] | Umbler`
  - Ex: `SDR AI — Como triplicar o atendimento sem contratar | Umbler`
- **Meta description:** o que o caso faz + resultado quantificado (≤155 chars)

## Padrão narrativo

```
1. Hero: resultado como headline
   "Nunca mais perca um lead por demora."

2. Antes/Depois: como é sem Umbler → como é com Umbler
   (Bullets curtos, visual lado a lado)

3. Mecanismo: como funciona (Steps 01/02/03)
   "Agente IA conecta no WhatsApp → qualifica → distribui no CRM"

4. Prova: métricas, social proof, screenshots do uso real

5. CTA: trial + demo dedicada
```

## Verbos concretos

Use Case **vive de verbos**. O leitor precisa visualizar **o que o produto faz** no caso dele:

> responde · qualifica · agenda · encaminha · distribui · organiza · alerta ·
> follow-up · escala · prioriza · reativa · converte

Anti-pattern: descrições com substantivos abstratos ("solução de qualificação automatizada"). O caso some.

## Diferença Use Case vs. Produto

| | Produto | Use Case |
| --- | --- | --- |
| Pergunta que responde | "O que esse produto faz?" | "Como esse produto resolve X?" |
| Escopo | Produto inteiro | Uma aplicação específica |
| CTA dominante | Trial geral | Trial + demo dedicada (mais qualificado) |
| Sequência | Hero + Features + Pricing | Hero + Antes/Depois + Mecanismo + Prova |

Um produto pode ter **vários** use cases. Cada um vira sua LP, porque cada um tem dor própria, linguagem própria, persona própria.

## Exemplos de bons use cases (categorias)

- **SDR AI** — agente que faz o trabalho de pré-vendas
- **Qualificação de Leads** — separar lead bom de lead ruim
- **Agendamento Automático** — bot que marca reunião no calendar
- **Disparo em Massa WhatsApp** — campanha sem virar spam
- **Multi-atendimento** — equipe inteira no mesmo número
- **Tráfego Pago → Conversas** — Meta Ads conectado direto no WhatsApp
- **Reengajamento de Leads** — recuperar lead frio

Cada um responde uma pergunta diferente. Cada um merece sua LP.

## Anti-patterns

- Use Case que vira "feature list" do produto inteiro (perdeu foco no caso)
- Sem antes/depois — falha o "JTBD funcional" (job to be done)
- Métricas inventadas / sem cliente concreto que validou
- CTA "Experimente o produto" em vez de "Agende demo do use case"
