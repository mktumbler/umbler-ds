Free Tool é a página onde **a ferramenta funciona ali na hora**. Não descreve a tool — entrega.

<Metodologia title="Por que a tool entrega antes de vender">

O resto da página (FAQ, social proof, upsell) é apoio à conversão pra produto pago. A tool é **gancho** pra produto pago: quem chega lá e usa precisa saber que existe algo maior — mas sem ser empurrado. Free tool **não substitui produto**. Complementa — atrai topo de funil que ainda não está pronto pra comprar. Cada free tool resolve **uma micro-dor** que o ICP da Umbler tem.

</Metodologia>

## Objetivo

Visitante sai com:
- **Resultado obtido** (o output da ferramenta — link gerado, análise, conversão)
- **Marca registrada** na cabeça como útil
- **Caminho sutil** para o produto pago (upsell discreto, não agressivo)

## Seções opcionais

- **FAQ** — perguntas sobre uso da ferramenta + limites
- **How It Works** — 3 passos pra usar (se a tool tiver fluxo)
- **Social Proof** — número de uso ("já gerou X links")

## SEO

- **Meta title:** `[Nome da tool] — [O que faz] | Grátis`
  - Ex: "contate.me — Gerador de Link de WhatsApp | Grátis"
- **Meta description:** o que faz + grátis + sem cadastro (≤155 chars)
- **Schema.org:** `WebApplication` ou `Service`

## Padrão narrativo

```
1. Hero curtíssimo (3-4 linhas)
   - O nome da tool
   - O que ela faz em 1 frase
   - Input/CTA acima da dobra

2. Tool Interface (funcionando)
   - Sem cadastro pra primeiro uso
   - Resultado imediato

3. Feature bullets — "o que faz" / "o que não faz"
   (Clareza honesta sobre limites)

4. [opcional] FAQ + How It Works

5. CTA upsell para Umbler
   "Gostou? A Umbler tem isso + dezenas de outras ferramentas pra crescer no WhatsApp"
   (Subtle, mas presente)
```

## Princípios

- **A tool funciona** — sem form pra "liberar" o uso; captura de email depois do uso, opcional
- **Mobile-first, sempre** — layout ruim no celular mata a tool
- **Fast** — pesa pouco, carrega rápido
- **Upsell sutil mas presente** — o CTA de upsell aparece **depois** do uso da tool, não antes

> "Gostou do contate.me? A Umbler organiza todas as conversas que ele gera, em um CRM que entende WhatsApp."

<Metodologia title="Por que esses princípios decidem o destino da tool">

**A tool funciona.** Não pode ser "preencha o form pra liberar o gerador". O leitor chega → usa → sai com resultado.

**Mobile-first.** Free tools são tipicamente buscadas via mobile (alguém precisa do output rápido em movimento). Layout que funciona ruim no celular **mata a tool**.

**Fast.** Free tool com 5s de loading vs. concorrente que carrega em 1s — o concorrente ganha.

**Upsell sutil.** Quem usa a tool precisa saber que existe algo maior, mas sem ser empurrado no meio do uso.

</Metodologia>

## Exemplos de free tools

- **Gerador de link WhatsApp** (contate.me) — usuário cola número + mensagem, recebe link encurtado
- **Calculadora de ROI de SDR** (click.now) — usuário coloca métricas, recebe estimativa
- **Auditor de WhatsApp Business** — analisa configuração de conta WAB

## Diferença Free Tool vs. Produto

| | Produto | Free Tool |
| --- | --- | --- |
| O que entrega | Plataforma completa | Uma micro-função |
| Custo | Pago, com trial | Grátis, sem cadastro |
| Persona | Decisor (compra software) | Operador (resolve problema agora) |
| Conversão | Trial → assinatura | Awareness → caminho pra produto |
| LP | Heavy (features, prova, FAQ extenso) | Leve (tool funciona ali) |

## Anti-patterns

- Form longo antes de poder usar a tool
- Tool com loading lento ou bugs (queima a marca)
- Upsell agressivo no meio do uso ("Pra continuar, assine!")
- Hero institucional ("Umbler te ajuda a crescer") em LP de tool simples
- Falta de mobile responsiveness (tool inutilizável no celular)
- Sem CTA pra produto pago em lugar nenhum (perde o upsell)
