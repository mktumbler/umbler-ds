Conversion não é só lógica. Métricas confirmam, mas a decisão é emocional — e essa página é como **costurar emoção** sem cair em pieguice. A marca pode soltar a ambição aqui (5% de humor, 20% personalidade da mistura ideal).

## O padrão Manifesto

Seção puramente tipográfica que cria **conexão emocional** entre seções feature-heavy. Funciona como pausa narrativa.

### Como compor

- **Texto bold** (forte, convicto) alternado com **texto muted** (explicativo, empático)
- Tipografia grande, **respiração generosa**, alinhamento centrado ou esquerda
- Opcionalmente termina com citação atribuída (nome + cargo + empresa)
- Sem ícones, sem CTAs, sem imagens — texto puro

### Por que funciona

O **Peak-End Rule** (Kahneman): a percepção geral de uma experiência é determinada pelo momento mais emocionalmente intenso + o momento final. O manifesto cria o pico emocional da LP. Quando o leitor chegar no CTA depois, vai estar "preparado emocionalmente" para converter.

### Onde posicionar

Após 2-3 seções de features/informação, antes de pricing/CTA. Serve como **respiro e reconexão emocional** antes da decisão de compra.

**Anti-pattern.** Manifesto logo no início da LP (não há contexto pra emoção). Ou manifesto sem ponto de chegada (vira poesia perdida).

## Yes Ladder

A página inteira é uma **escada de micro-agreements**. Cada seção busca um "sim" mental que facilita o próximo:

```
1. "Sim, eu tenho esse problema"        (dor reconhecida)
2. "Sim, isso parece uma solução"       (mecanismo aceito)
3. "Sim, outras empresas confiam"        (prova validada)
4. "Sim, isso se aplica ao meu caso"     (relevância confirmada)
5. "Sim, o preço é justo"                (valor percebido)
6. "Sim, minhas objeções foram respondidas" (resistência removida)
7. "Sim, vou experimentar"               (ação tomada)
```

**Como aplicar.** Para cada seção da LP, identifique qual "sim" ela busca. Se uma seção não corresponde a nenhum sim — sai. Se duas seções buscam o mesmo sim — combinam ou cortam uma.

## Tension-Release

Já tratado em [Estrutura](/docs/marketing/conversion/estrutura#tension-release-rhythm). Aqui o ponto operacional: **toda seção é tensão OU alívio**, nunca neutra. Neutra é seção que pode sair sem perda.

Quando você sentir que a LP está "monótona", quase sempre é falta de alternância. Adicione uma seção de tensão (uma dor não dita, um risco real) ou de alívio (um caso concreto, uma prova).

## Persuasão visual

Design **comunica antes da palavra ser lida**. Três aplicações:

### Whitespace como persuasão

Espaçamento generoso comunica **qualidade premium**. Marcas baratas comprimem tudo; marcas premium deixam respirar.

- Padding ao redor de CTAs **aumenta cliques** porque reduz ruído cognitivo
- Margens entre seções dão tempo de "absorver" antes do próximo argumento
- Cards com espaço interno generoso parecem mais valiosos que cards apertados

Compre tempo de leitura com espaço, não com texto.

### Wow moments

Animações premium (word reveals, parallax, counters animados, shine borders) criam a percepção de **"esta empresa investe em excelência"**. Isso se transfere inconscientemente para o produto.

**Regra crítica.** Máximo **2-3 wow moments por página**. Mais que isso vira fadiga visual e a percepção inverte — "essa empresa está se esforçando demais pra parecer boa".

Onde colocar os wow moments:
1. **Hero** (entrada — uma animação de impacto)
2. **Manifesto** (pico emocional — reveal sutil, gradient progression)
3. **CTA banner final** (saída — um detalhe que fecha)

### Hierarquia visual = hierarquia de mensagem

O elemento mais importante na página é o **maior + mais contrastado + mais isolado**. Se o headline está fraco visualmente, o leitor não vai chegar nele.

**Sinal de problema.** Hero onde 3 elementos competem pela mesma força (logo gigante + headline gigante + visual gigante). O olho não sabe pra onde ir, e some.

## A regra final

Storytelling **complementa** a lógica, nunca substitui. Uma LP que é só emoção (manifesto, gradients, wow moments) e nenhuma prova/feature vira teaser. Uma LP que é só lógica (features, tabela, FAQ) sem manifesto vira spec sheet.

Mistura: 70% lógica (features, prova, sequência, CTA) + 30% storytelling (hero emocional, manifesto, ritmo visual). Esse 30% é o que faz o leitor **se importar** com os 70%.
