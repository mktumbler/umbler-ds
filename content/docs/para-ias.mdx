Você não precisa decorar fluxos. Cada cenário é **um comando + falar com a IA em PT**.

## Os três caminhos

### 1. Criando uma LP nova

```bash
npx shadcn@latest init
npx shadcn@latest add https://umbler-ds.vercel.app/r/umbler-ui
```

Instala tokens + utilitário + **todos os componentes** de uma vez. Blocks (hero, pricing, FAQ, CTA…) você pede por nome quando precisar — a IA roda `shadcn add` da URL certa por baixo.

Depois, **cole no contexto da sua IA** (Cursor rule, Custom Instructions, system prompt):
> `https://umbler-ds.vercel.app/UMBLER.md`

Pronto. Agora é só pedir: *"Cria uma LP de [produto] com hero, pricing e FAQ"*.

### 2. Projeto já existente

Mesma coisa, sem o `init` (você já tem shadcn ou Tailwind v4):

```bash
npx shadcn@latest add https://umbler-ds.vercel.app/r/umbler-ui
```

E o mesmo passo da IA: cole `https://umbler-ds.vercel.app/UMBLER.md` na regra/instruções permanentes.

<Callout type="warn" title="Pré-requisito">
Tailwind v4 e Next.js (App Router). Em v3 não funciona — os tokens usam `@theme` do v4.
</Callout>

### 3. Usando Claude Code

Um comando, dispensa o UMBLER.md (o plugin já carrega a skill):

```
/plugin add umbler-ds
```

A partir daí, fale em PT. *"Cria uma LP com hero e FAQ"* — o Claude reconhece os blocks, roda `/umbler-add` pra cada um e compõe a página. Auditoria de hand-rolls roda como hook automático em cada arquivo editado.

[Detalhes do plugin →](/docs/plugin)

## Por que isso funciona

| Arquivo | O que carrega na IA |
|---|---|
| [`/UMBLER.md`](https://umbler-ds.vercel.app/UMBLER.md) | Voz, tokens, regras, antipatterns — para qualquer IA fora do Claude Code |
| [`/llms.txt`](https://umbler-ds.vercel.app/llms.txt) | Catálogo técnico de componentes com props — referência rápida pra gerar TSX |
| [Plugin Claude Code](/docs/plugin) | Skill + slash commands + auto-audit — pra quem usa Claude Code |

UMBLER.md e llms.txt são públicos e estáticos. **Você nunca precisa atualizar nada** — o DS regenera ambos a cada build e os mantém em sincronia.

## E se eu não usar IA?

Funciona igual. `shadcn add` instala o componente, você importa e usa:

```tsx
import { Button } from '@/components/ui/button';
import { HeroBlock } from '@/components/blocks/hero-block';
```

A IA acelera, não é pré-requisito.
