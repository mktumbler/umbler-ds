O Umbler DS é a base de marca da Umbler: **design tokens**, **componentes** e **blocks** prontos para montar **landing pages**, **e-mails** e **produtos** — todos no mesmo padrão.

Tudo é distribuído via registry `shadcn`: o código é **copiado** para o seu projeto, sem dependência de runtime e totalmente editável. **Funciona melhor com IA** — Cursor, ChatGPT, Claude, Windsurf, Gemini, Copilot. Você descreve em PT, a IA monta no padrão.

<Callout title="Pré-requisito">
Projeto **Next.js (App Router)** com **Tailwind CSS v4** e **TypeScript**. Os tokens usam `@theme` do Tailwind v4 — **não funciona no v3**.
</Callout>

## Dois caminhos

### Caminho A — Claude Code (recomendado)

Um comando. A skill já carrega tokens, voz, regras e slash commands:

```
/plugin add umbler-ds
```

Depois fale em PT: *"Cria uma LP do produto X com hero, pricing e FAQ no padrão Umbler"*. O plugin reconhece os blocks, instala sob demanda, audita hand-rolls a cada `.tsx` editado, e dispensa qualquer outro passo.

[Detalhes do plugin →](/docs/plugin)

### Caminho B — Qualquer outra IA (Cursor, ChatGPT, Windsurf, Gemini, Copilot…)

**1. Instale o DS** (no terminal do projeto):

```bash
npx shadcn@latest init                                            # só se ainda não tem shadcn
npx shadcn@latest add https://umbler-ds.vercel.app/r/umbler-ui    # todos os componentes
```

**2. Carregue o briefing da marca na sua IA.** Cole esta URL nas instruções permanentes (Cursor rule, Custom Instructions do ChatGPT, system prompt):

```
https://umbler-ds.vercel.app/UMBLER.md
```

**3. Peça em PT.** *"Cria uma LP do produto X com hero, pricing e FAQ no padrão Umbler"* — a IA reconhece os blocks, instala o que faltar e compõe.

<Callout type="warn" title="Um único ajuste depois do shadcn init">
O `init` injeta um tema concorrente. Como o DS já traz o sistema completo, deixe seu `app/globals.css` apenas com:

```css
@import 'tailwindcss';
@import './tokens.css';
```
</Callout>

## Recursos pra IA

| Recurso | Pra quê |
|---|---|
| [`/UMBLER.md`](https://umbler-ds.vercel.app/UMBLER.md) | **Briefing de marca**: voz, tokens, regras, antipatterns. Cola na instrução de qualquer IA. |
| [`/llms.txt`](https://umbler-ds.vercel.app/llms.txt) | **Catálogo técnico**: componentes com props e exemplos. Pra IA gerar TSX preciso. |
| [Plugin Claude Code](/docs/plugin) | Skill + `/umbler-add` + auto-audit. Pra quem usa Claude Code. |

Ambos os arquivos são **regenerados automaticamente** a cada build do DS — você nunca precisa atualizar nada do seu lado.

## E se eu não usar IA?

Funciona igual. `shadcn add` instala o componente, você importa e usa:

```tsx
import { Button } from '@/components/ui/button';
import { HeroBlock } from '@/components/blocks/hero-block';
```

A IA acelera, não é pré-requisito.
