O Umbler DS é uma coleção de **tokens de marca**, **componentes** e **blocks de landing page** distribuída via registry shadcn. O código é copiado para o seu projeto — sem dependência de runtime, totalmente editável.

O uso principal é gerar **landing pages Umbler completas** — hero, features, pricing, FAQ, CTA — no padrão correto de marca, copy e conversão.

<Callout title="Pré-requisito">
Projeto **Next.js (App Router)** com **Tailwind CSS v4** e **TypeScript**. Os tokens usam `@theme` do Tailwind v4 — não funciona no v3.
</Callout>

## Passo 1 — Instale os blocks de LP

Um comando instala todos os 10 blocks necessários para landing pages:

```bash
npx shadcn@latest add https://umbler-ds.vercel.app/r/lp-kit.json
```

Isso copia Hero, Pricing, FAQ, CTA Banner, Stat Grid, Feature Cards, Feature Split, Testimonial, Logo Cloud e Site Footer para `components/blocks/` do seu projeto.

## Passo 2 — Gere a LP

### Com Claude Code

Copie a pasta `.claude-plugin/` do [repositório do DS](https://github.com/mktumbler/umbler-ds) para o seu projeto e use o comando:

```
/umbler-lp talk
/umbler-lp mail
/umbler-lp hosting
```

O comando lê o catálogo de produtos (persona, dor, pricing, FAQ de cada produto Umbler) e gera a página completa usando os blocks instalados no passo 1. Produto não está no catálogo? O comando faz 3 perguntas e gera assim mesmo.

### Com qualquer outra IA (Cursor, ChatGPT, Gemini…)

Carregue o briefing de marca no contexto permanente da IA:

```
https://umbler-ds.vercel.app/UMBLER.md
```

Depois use este prompt:

```text
Crie uma landing page para [produto] no padrão Umbler.
Estrutura: hero + features + pricing (3 planos) + FAQ (5 perguntas) + CTA final.
Persona: [quem é o cliente]. Dor principal: [problema que resolve]. CTA: [ação + URL].
Os blocks estão em components/blocks/ — importe direto, sem reinstalar.
```

## Outros prompts prontos

**E-mail transacional:**

```text
Crie um e-mail de [boas-vindas | recuperação de senha | confirmação] no padrão
Umbler, usando os templates react-email do DS. Tom mentor, sem em-dash.
```

**Aplicar identidade em material existente:**

```text
Tenho [apresentação | banner | documento] sobre [tema]. Reescreva a copy no
tom Umbler (mentor, sem jargão, sem em-dash) e aplique os tokens de cor e
tipografia da marca. Liste os tokens que usou.
```

## Recursos para IA

| Recurso | Para quê |
|---|---|
| [`/UMBLER.md`](https://umbler-ds.vercel.app/UMBLER.md) | Briefing de marca: voz, tokens, regras, antipatterns. Cole na instrução permanente de qualquer IA. |
| [`/llms.txt`](https://umbler-ds.vercel.app/llms.txt) | Catálogo técnico: todos os componentes com props e exemplos de uso. |

Ambos são regenerados automaticamente a cada build do DS — você nunca precisa atualizar nada do seu lado.
