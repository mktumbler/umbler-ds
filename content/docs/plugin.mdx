O Umbler DS é distribuído também como **plugin do Claude Code**. Quem instala, ganha:

- O Claude já sabe **todas as convenções** do DS (não precisa explicar)
- **2 slash commands**: instalar (com bootstrap automático) e auditar
- **Hook automático** que avisa de hand-rolls toda vez que você edita `.tsx` ou `.mdx`

## Instalação

No Claude Code, abra **Diretório → Plugins → Sua organização** e procure **Umbler DS**.

Ou via comando:

```
/plugin add umbler-ds
```

## Slash commands

| Comando | Função |
|---|---|
| `/umbler-add <componente>` | Adiciona um componente ou block da registry e valida adoção. Se o projeto ainda não tem o DS, faz o bootstrap automaticamente (`shadcn init` + tokens) |
| `/umbler-audit` | Varre o repo atrás de hand-rolls (button/badge/eyebrow/alert) |

O fluxo de criação de blocos (Interface Inventory + regra do 3) vive na **skill** — peça "quero criar um bloco novo" e o Claude conduz.

## Hook

`PostToolUse` em `Edit|Write` dispara `audit-antipatterns.mjs` em modo warning. Não bloqueia a edição — só avisa se você ou o Claude escreveram algo que duplica um componente do DS.

## Como o plugin sabe o que existe

A skill `umbler-ds` está configurada para baixar `https://umbler-ds.vercel.app/llms.txt` (catálogo) via WebFetch antes de gerar UI. Significa: **você não precisa atualizar o plugin quando um componente novo é adicionado ao DS** — o catálogo é sempre fresh.

## Estrutura interna

```
.claude-plugin/
├── plugin.json          ← manifest
├── skills/umbler-ds/    ← contexto + regras
├── commands/            ← 2 slash commands
├── hooks/               ← auto-audit
└── scripts/             ← audit portátil
```

## Quando o plugin **não** é suficiente

Se você é mantenedor do DS (não apenas consumer), use direto o repo `umbler-ds` — você precisa de acesso ao código-fonte dos componentes, demos, MDX e registry. O plugin é otimizado para **consumir** o DS, não evoluí-lo.

## Versionamento

O plugin acompanha versões do DS. Se você fixou em `v0.1.0` e o DS evoluiu para `v0.2.0`, o catálogo via WebFetch continua mostrando os novos componentes — mas slash commands antigos seguem funcionando como antes. Atualize o plugin quando quiser ganhar novos comandos ou skills.
